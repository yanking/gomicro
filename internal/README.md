# 内部包

此目录包含应用程序的内部包。这些包不打算供外部应用程序使用。

## 目录结构

- `config/` - 配置管理
- `handler/` - HTTP 处理程序
- `service/` - 业务逻辑服务
- `repository/` - 数据访问层
- `model/` - 领域模型
- `server/` - HTTP 服务器设置
- `logger/` - 日志工具
- `middleware/` - HTTP 中间件
- `utils/` - 工具函数

## 包组织

每个包应遵循单一职责原则，并具有明确的目的。包之间不应存在循环依赖。

## 依赖注入

包之间的依赖应通过接口注入，以促进松耦合和可测试性。
